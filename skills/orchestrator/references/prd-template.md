# PRD Template

```markdown
# PRD: {Feature Name}

**Date**: {YYYY-MM-DD}
**Status**: Draft | Approved
**Author**: Claude (Full Auto)

## Summary
One paragraph: what this feature does and why it matters.

## User Stories
- As a {role}, I want {capability} so that {benefit}

## Acceptance Criteria
- [ ] Criterion with measurable outcome

## Non-Goals
- What this feature explicitly does NOT do

## Technical Constraints
- Required frameworks, patterns, or APIs
- Performance requirements
- Compatibility requirements
- Existing code patterns to follow

## Dependencies
- External libraries or services needed
- Other features that must exist first

## Test Strategy
- Unit tests: what to test
- Integration tests: what to test
- Manual verification: steps to confirm it works

## Open Questions
- Must be empty before advancing to Phase 2
```
